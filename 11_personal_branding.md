## 二、平台选择

### 2.1 内容平台对比

| 平台 | 优势 | 劣势 | 适合内容 |
|------|------|------|---------|
| **Twitter/X** | 传播快，互动强 | 内容碎片化 | 短思考，动态 |
| **LinkedIn** | 专业，B2B | 较正式 | 职业动态，深度文 |
| **掘金** | 中文技术社区 | 流量有限 | 技术文章 |
| **知乎** | 长尾流量 | 商业化弱 | 问答，专栏 |
| **公众号** | 私域流量 | 封闭生态 | 深度文章 |
| **YouTube** | 视频化 | 制作成本高 | 教程，演示 |
| **Blog** | 完全掌控 | 需自己引流 | 深度内容 |

### 2.2 推荐组合

```
入门组合（选 2-3 个）：
- Twitter + 掘金 + GitHub
  （国际曝光 + 中文社区 + 代码展示）

进阶组合：
- Twitter + LinkedIn + Blog + YouTube
  （全方位覆盖）

专注组合：
- 公众号 + 知乎
  （深耕中文市场）
```

## 五、写作技巧

### 5.1 文章结构

```markdown
## 标准技术文章结构

# 标题（吸引人）

## 前言
- 背景介绍
- 为什么要写这个
- 读者能获得什么

## 正文
### 1. 问题描述
### 2. 解决方案
### 3. 实现细节
### 4. 代码示例

## 总结
- 核心要点回顾
- 下一步建议

## 参考资料
- 相关链接
```

### 5.2 写作要点

```
✅ 好文章特点：
1. 标题吸引人但不过度
2. 开头抓住注意力
3. 结构清晰有层次
4. 代码示例完整
5. 结论明确有用

❌ 避免：
1. 标题党
2. 冗长啰嗦
3. 缺少示例
4. 过时信息
5. 纯理论无实践
```

## 八、常见问题

### Q1: 没时间输出怎么办？

**A:** 批量创作，定时发布，降低频率但保证质量

### Q2: 不知道写什么？

**A:** 记录学习过程，解答常见问题，分享项目经验

### Q3: 增长缓慢怎么办？

**A:** 坚持输出，优化内容，跨平台引流
